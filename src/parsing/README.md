# VCF Parsing — Column Reference for Rare Disease GWAS

This document is a complete reference for every column produced by `parse_vcf.py`.
It is written for **data scientists** who may not have a deep bioinformatics background but need to
understand and correctly filter the parsed variant data for downstream analyses such as burden tests,
association studies, or machine learning models applied to rare disease cohorts.

The parser reads **VEP-annotated VCF files** generated by the Sarek pipeline
(DeepVariant caller + Ensembl VEP v113 annotation, GRCh38 reference genome) and flattens each
variant into one row per patient.

---

## Background: What is a VCF File?

A **VCF (Variant Call Format)** file is the standard text file for storing differences between
an individual's genome and a reference genome. Each row in the data section of a VCF file
represents one **genomic position** where the sample differs from the reference.

The reference genome (here, **GRCh38 / hg38**) is a composite "average" human genome used as a
coordinate system. Every position in a VCF file is described relative to it.

Variants come in several flavours:

| Type | Abbreviation | Example |
|------|-------------|---------|
| Single nucleotide variant | SNV | REF=A, ALT=G |
| Small insertion | INS | REF=A, ALT=ATCG |
| Small deletion | DEL | REF=ATCG, ALT=A |
| Multi-nucleotide variant | MNV | REF=AT, ALT=GC |

After variant calling, each variant is **annotated** by Ensembl VEP, which predicts the
biological effect of each change (e.g., does it alter a protein?) and looks up its frequency in
large population databases such as gnomAD.

## 1. Standard VCF Fields

These seven columns plus `Patient_ID` are the core of every VCF file and carry information
produced by the **variant caller** (DeepVariant), before any biological annotation is applied.

---

### `Patient_ID`
**Type:** string

The stem of the VCF filename (filename without the `.vcf` extension). It is used as the unique
patient identifier throughout the analysis. Because the pipeline produces one VCF per sequenced
sample, the filename is the most reliable key.

Example: a file named `Case1_F.vcf` produces `Patient_ID = "Case1_F"`.

---

### `CHROM`
**Type:** string

The name of the chromosome on which the variant was found. Human chromosomes are numbered
1–22 (autosomes), X and Y (sex chromosomes), and M (mitochondrial genome). In GRCh38 VCFs
the prefix `chr` is always present.

Standard values: `chr1`–`chr22`, `chrX`, `chrY`, `chrM`.

Unplaced/alternative contigs (e.g., `chrUn_KI270302v1`) represent regions of the genome that
could not be placed on a specific chromosome in the assembly. Variants on these contigs are
typically excluded from standard GWAS analyses.

---

### `POS`
**Type:** integer

The **1-based** genomic coordinate of the variant on the chromosome. "1-based" means the first
nucleotide of a chromosome is position 1 (unlike 0-based BED format where it would be 0).

For insertions, POS points to the last reference base before the inserted sequence.
For deletions, POS points to the first deleted base.

---

### `ID`
**Type:** string | None

A known identifier for the variant, most commonly an **rsID** from the NCBI dbSNP database
(e.g., `rs376007522`). The `rs` prefix stands for "Reference SNP cluster ID."

`None` (`.` in the raw file) means the variant has not been catalogued in dbSNP at this position
and may be a **novel variant** — particularly important in rare disease research, where pathogenic
mutations are often rare or even family-private.

A single position can have multiple rsIDs (comma-separated) if more than one variant has been
reported there in the databases.

---

### `REF`
**Type:** string

The **reference allele** — the sequence present at this position in the GRCh38 reference genome.
This is always one or more nucleotide characters (A, C, G, T). For SNVs it is a single base;
for deletions it is the deleted sequence plus one flanking base.

> Note: `REF` and `ALT` together define the variant. A change from `REF=C` to `ALT=T` is a
> cytosine-to-thymine substitution; `REF=ACGT`, `ALT=A` is a deletion of three bases.

---

### `ALT`
**Type:** string

The **alternate allele** — the sequence actually observed in this patient at this position.
Multiple alternate alleles are comma-separated (multi-allelic site), though after splitting
multi-allelic records each row has a single `ALT`.

Symbolic alleles like `<DEL>` or `<INS>` indicate large structural variants; these are
not handled by this parser and should be filtered out before analysis.

---

### `QUAL`
**Type:** float | None

The **Phred-scaled quality score** of the variant call, expressing the caller's confidence that
this is a true variant and not a sequencing error.

The Phred scale is logarithmic:

| QUAL | Probability that the call is WRONG |
|------|-------------------------------------|
| 10 | 10 % (1 in 10) |
| 20 | 1 % (1 in 100) |
| 30 | 0.1 % (1 in 1000) |
| 50 | 0.001 % (1 in 100 000) |

DeepVariant can emit `QUAL = 0` for `RefCall` sites (where the caller prefers the reference
allele), which should be interpreted as a low-confidence or absent variant rather than a
confirmed reference call.

`None` is stored when the raw field is `.` (missing).

---

### `FILTER`
**Type:** string

Pass/fail label(s) assigned by the variant caller after applying internal thresholds.
For downstream analyses, **only `PASS` variants should be used**.

| Value | Meaning |
|-------|---------|
| `PASS` | The variant passed all internal quality filters — the caller is confident this is a real variant |
| `RefCall` | The genotyping model prefers the reference allele; the alternate allele evidence is weak |
| `LowQual` | Confidence in the variant being real falls below the caller's calling threshold |
| `NoCall` | The site has zero read depth — no call can be made |

Multiple filter labels can appear semi-colon-separated when more than one filter is triggered.

## 2. VEP / CSQ Annotation Fields

After variant calling, each variant is passed through **Ensembl VEP (Variant Effect Predictor)
v113**, which annotates the biological consequences of each variant. VEP stores all annotations
in a single INFO sub-field called `CSQ` (Consequence), pipe-delimited.

The pipeline uses the `--pick` flag, which instructs VEP to select **one canonical transcript
per variant** (the most biologically relevant one, based on a priority ranking). The result is
one annotation row per variant per patient.

---

### 2a. Transcript & Gene Identity

These fields tell you *which gene* is affected and via *which transcript*.

---

#### `SYMBOL`
**Type:** string | None

The **HGNC gene symbol** — the short, standardised human-readable name for a gene, assigned by
the HUGO Gene Nomenclature Committee. This is the name you would look up in a textbook or
clinical report (e.g., `BRCA2`, `TP53`, `CFTR`).

Gene symbols are stable but can occasionally be updated when gene function is reclassified.
For programmatic joins across databases, prefer the Ensembl Gene ID (`Gene`) which never changes.

`None` for intergenic variants that do not overlap any annotated gene.

---

#### `Gene`
**Type:** string | None

The **Ensembl stable gene identifier** (e.g., `ENSG00000139618`). This is the most reliable
cross-database key for a gene because it does not change when a gene is renamed.

Format: `ENSG` followed by 11 digits. The prefix `ENSG` is specific to human genes annotated
by Ensembl.

Use this column for programmatic lookups in databases such as Ensembl BioMart, STRING, or
when building gene × patient burden matrices.

---

#### `Feature_type`
**Type:** string | None

The type of genomic feature that the variant overlaps. Most variants in coding analyses will
have `Transcript`. Other types are relevant for non-coding regulatory analyses.

| Value | Meaning |
|-------|---------|
| `Transcript` | The variant overlaps an annotated gene transcript (coding or non-coding) |
| `RegulatoryFeature` | The variant overlaps a regulatory element (promoter, enhancer, etc.) annotated by Ensembl Regulatory Build |
| `MotifFeature` | The variant overlaps a transcription factor binding motif |

---

#### `Feature`
**Type:** string | None

The **Ensembl stable transcript identifier** (e.g., `ENST00000380152`) when `Feature_type` is
`Transcript`. A single gene can have multiple transcripts (alternative splicing), and `--pick`
selects the canonical one.

Format: `ENST` followed by 11 digits. Stable across Ensembl releases.

---

#### `BIOTYPE`
**Type:** string | None

The functional classification of the transcript selected by VEP. This is critical for
filtering: most variant effect analyses focus on `protein_coding` transcripts.

| Value | Meaning | Relevance for rare disease |
|-------|---------|--------------------------|
| `protein_coding` | Transcript that encodes a protein | Primary target for GWAS / burden testing |
| `lncRNA` | Long non-coding RNA | Can be disease-relevant, especially in regulatory disorders |
| `processed_pseudogene` | Retrotransposed copy of a gene, usually not translated | Generally excluded |
| `miRNA` | MicroRNA — regulates gene expression post-transcriptionally | Exclude unless studying regulatory mechanisms |
| `retained_intron` | Transcript that retains an intron and is likely degraded by NMD | Usually excluded |
| `nonsense_mediated_decay` | Transcript predicted to be degraded | Usually excluded |

For standard rare-disease coding variant analyses, filter to `BIOTYPE == "protein_coding"`.

---

#### `SYMBOL_SOURCE`
**Type:** string | None

The database that provided the gene symbol in `SYMBOL`.

| Value | Meaning |
|-------|---------|
| `HGNC` | Official HUGO Gene Nomenclature Committee symbol |
| `EntrezGene` | Symbol from NCBI Entrez Gene database |
| `Clone_based_ensembl_gene` | Symbol derived from clone-based naming — often for poorly annotated loci |

---

#### `HGNC_ID`
**Type:** string | None

The unique numeric identifier from the HGNC database, always prefixed with `HGNC:` (e.g.,
`HGNC:1101`). This is useful for unambiguous lookups of gene information in HGNC, Orphanet,
OMIM, and similar databases.

---

#### `SOURCE`
**Type:** string | None

The annotation source for the transcript. Ensembl and RefSeq use different transcript numbering
systems and may disagree on transcript boundaries.

| Value | Meaning |
|-------|---------|
| `Ensembl` | Transcript from Ensembl/GENCODE annotation |
| `RefSeq` | Transcript from NCBI RefSeq annotation |

---

### 2b. Variant Effect Classification

These fields are the core biological output of VEP: they tell you *what effect* the variant
has on the gene product.

---

#### `Allele`
**Type:** string | None

The alternate allele as VEP represents it internally. For simple SNVs this equals `ALT`.
For complex variants (e.g., multi-nucleotide polymorphisms or indels), VEP may normalise or
left-align the allele differently from the raw `ALT`. Use `HGVSc`/`HGVSp` for unambiguous
variant description.

---

#### `Consequence`
**Type:** string | None

A **Sequence Ontology (SO)** term describing the predicted molecular effect of the variant on
the selected transcript. Multiple effects can appear slash-separated when a single variant
triggers more than one consequence (e.g., `missense_variant&splice_region_variant`).

This is the most important annotation field for variant prioritisation. See the full ranked list
in [Section 5](#5-consequence-severity-ladder).

Commonly encountered values in rare disease:

| Consequence | Plain English |
|-------------|---------------|
| `missense_variant` | The variant changes one amino acid to a different amino acid |
| `synonymous_variant` | The variant changes a codon but the amino acid stays the same (silent) |
| `stop_gained` | The variant introduces a premature stop codon, truncating the protein |
| `frameshift_variant` | An insertion or deletion shifts the reading frame, changing all downstream amino acids |
| `splice_donor_variant` | The variant disrupts the GT splice signal at the start of an intron |
| `splice_acceptor_variant` | The variant disrupts the AG splice signal at the end of an intron |
| `splice_region_variant` | The variant is near (but not in) a splice site; may affect splicing |
| `inframe_insertion` | An insertion that is a multiple of 3 bp — adds amino acid(s) without shifting the frame |
| `inframe_deletion` | A deletion that is a multiple of 3 bp — removes amino acid(s) without shifting the frame |
| `start_lost` | The variant disrupts the ATG start codon — translation cannot initiate |
| `stop_lost` | The variant changes the stop codon to a sense codon — the protein is extended |
| `upstream_gene_variant` | The variant is within 5 kb upstream of the gene (not in the gene itself) |
| `intron_variant` | The variant falls inside an intron |
| `3_prime_UTR_variant` | The variant is in the 3′ untranslated region |
| `5_prime_UTR_variant` | The variant is in the 5′ untranslated region |
| `intergenic_variant` | The variant is between genes, with no annotated feature overlap |

---

#### `IMPACT`
**Type:** string | None

A coarse four-tier severity classification assigned by VEP to each `Consequence`. It provides
a quick filter without needing to enumerate all SO terms.

| Tier | Plain English | Consequence examples |
|------|---------------|----------------------|
| `HIGH` | Expected to destroy or severely disrupt protein function | `stop_gained`, `frameshift_variant`, `splice_acceptor_variant`, `splice_donor_variant`, `start_lost`, `transcript_ablation` |
| `MODERATE` | May alter protein function, but the protein is likely still produced | `missense_variant`, `inframe_insertion`, `inframe_deletion` |
| `LOW` | Unlikely to significantly alter protein function | `synonymous_variant`, `splice_region_variant`, `stop_retained_variant` |
| `MODIFIER` | Non-coding variant or variant far from a gene; effect on function is generally unknown | `intron_variant`, `upstream_gene_variant`, `3_prime_UTR_variant`, `intergenic_variant` |

For rare disease analyses, the standard approach is to **start with `HIGH` and `MODERATE`** variants,
then decide whether to include `LOW` (splice region) variants based on the research question.
`MODIFIER` variants require additional evidence (e.g., regulatory annotations) before inclusion.

---

#### `EXON`
**Type:** string | None

The exon in which the variant falls, expressed as `<exon_number>/<total_exons>`.
For example, `5/27` means the variant is in exon 5 of a transcript that has 27 exons.

`None` when the variant is not in an exon (intronic, upstream, intergenic, etc.).

Exon numbering follows the transcript annotation and is 1-based.

---

#### `INTRON`
**Type:** string | None

The intron in which the variant falls, expressed as `<intron_number>/<total_introns>`.
For example, `3/26` means the variant is in intron 3 of a transcript with 26 introns.

`None` when the variant is not intronic.

Most intronic variants have `IMPACT = MODIFIER` and are filtered out in standard analyses,
unless they are within ~10 bp of a splice site (which VEP would flag as `splice_region_variant`).

---

### 2c. Positional Nomenclature (HGVS)

HGVS (Human Genome Variation Society) nomenclature provides **standardised, unambiguous
descriptions** of variants at the DNA and protein level. These are the internationally accepted
formats for clinical reporting, ClinVar submissions, and literature references.

---

#### `HGVSc`
**Type:** string | None

The **HGVS coding DNA nomenclature** — describes where and what the change is relative to
the transcript's coding sequence.

Format: `<transcript_id>.<version>:c.<change>`

Examples and how to read them:

| HGVSc | Meaning |
|-------|---------|
| `ENST00000380152.8:c.5946del` | In transcript ENST00000380152 (version 8), a deletion at coding position 5946 |
| `ENST00000269305.9:c.1A>G` | Substitution of A→G at position 1 of the CDS (start codon!) |
| `ENST00000380152.8:c.5946+1G>T` | Substitution in the intron, 1 bp after position 5946 (splice donor) |
| `ENST00000380152.8:c.5946-3C>A` | Substitution in the intron, 3 bp before position 5946 (near splice acceptor) |

The `c.` prefix indicates coding sequence. Intronic positions use `+` (downstream of exon) or
`-` (upstream of exon) notation.

`None` for variants that do not affect a coding transcript (e.g., intergenic variants).

---

#### `HGVSp`
**Type:** string | None

The **HGVS protein nomenclature** — describes the resulting amino acid change.

Format: `<protein_id>.<version>:p.<change>`

Examples and how to read them:

| HGVSp | Meaning |
|-------|---------|
| `p.Ser982Arg` | Serine (Ser/S) at position 982 changed to Arginine (Arg/R) — missense |
| `p.Ser982ArgfsTer22` | Frameshift starting at position 982; new reading frame introduces a stop codon 22 residues later |
| `p.Arg117Ter` | Arginine at position 117 changed to a stop codon (Ter) — nonsense mutation |
| `p.Lys58del` | Lysine at position 58 is deleted (in-frame deletion) |
| `p.(Ser982Arg)` | Parentheses indicate the protein change is predicted (not experimentally confirmed) |
| `p.=` | The protein is unchanged (synonymous variant) |

`None` for non-coding variants or when VEP cannot predict the protein consequence.

---

#### `cDNA_position`
**Type:** string | None

The position within the full cDNA sequence (including UTRs) where the variant occurs.
Unlike `CDS_position`, cDNA coordinates include the 5′ UTR before the start codon.

For intronic variants, the position is given with `+` or `-` offsets from the nearest exon
boundary (e.g., `1234+5`).

---

#### `CDS_position`
**Type:** string | None

The position within the **Coding DNA Sequence (CDS)**, i.e., from the first base of the start
codon (A of ATG = position 1) to the last base of the stop codon. This is the coordinate
used in `HGVSc` nomenclature.

`None` for non-coding variants (introns, UTRs, intergenic).

---

#### `Protein_position`
**Type:** string | None

The amino-acid position in the translated protein where the variant effect occurs. Position 1
is the first methionine (start codon). For deletions or insertions spanning multiple residues,
a range is given (e.g., `329-331`).

`None` for non-coding variants.

---

#### `Amino_acids`
**Type:** string | None

The reference and alternate amino acids separated by `/`, using the one-letter IUPAC code.

| Value | Meaning |
|-------|---------|
| `S/R` | Serine → Arginine (missense) |
| `K/-` | Lysine deleted (in-frame deletion) |
| `-/G` | Glycine inserted (in-frame insertion) |
| `S/` | Serine → stop codon (stop gained; alt side is empty) |

`None` for synonymous variants (no amino-acid change) and non-coding variants.

---

#### `Codons`
**Type:** string | None

The reference and alternate codons (three-nucleotide sequences) separated by `/`. The changed
nucleotide(s) are shown in **uppercase**; unchanged nucleotides are lowercase.

Example: `tCa/tGa` means the middle nucleotide changed from C to G, converting a Serine codon
(TCA) to a Serine-losing codon (TGA = stop).

`None` for non-coding variants.

---

### 2d. Existing Variants & Distance

---

#### `Existing_variation`
**Type:** string | None

A comma-separated list of identifiers for previously reported variants at or overlapping this
position, drawn from multiple databases:

| Prefix | Database | Purpose |
|--------|----------|---------|
| `rs…` | dbSNP | General variant catalogue; most common |
| `COSM…` | COSMIC | Cancer somatic mutations database |
| `RCV…` / `VCV…` | ClinVar | Clinical significance classifications |

If this field contains a ClinVar accession (`RCV` or `VCV`), the variant has a clinical
interpretation — it may be listed as pathogenic, benign, or of uncertain significance.
Cross-referencing against ClinVar is an important step in rare disease variant prioritisation.

`None` for novel variants not in any of these databases.

---

#### `DISTANCE`
**Type:** string | None

The distance in base pairs between the variant and the nearest annotated transcript, reported
for **upstream** and **downstream** variants that do not overlap any transcript.

A value of `1901` means the variant is 1,901 bp from the transcript boundary.
VEP reports upstream/downstream variants within a default window of 5,000 bp.

`None` for variants that overlap a transcript (exonic, intronic, UTR).

---

#### `FLAGS`
**Type:** string | None

Warning flags that VEP attaches to the transcript to indicate potential annotation quality issues.

| Flag | Meaning |
|------|---------|
| `cds_start_NF` | The CDS start (ATG start codon) was not found in the transcript annotation — HGVS coordinates may be unreliable |
| `cds_end_NF` | The CDS end (stop codon) was not found — similar caveat |
| `ncRNA` | The selected transcript is non-coding despite the gene being protein-coding |

Variants on transcripts with these flags should be interpreted with caution or re-annotated
against an alternative transcript.

---

### 2e. Gene Model Metadata

---

#### `STRAND`
**Type:** string | None

The genomic strand on which the feature (gene/transcript) is encoded.

| Value | Meaning |
|-------|---------|
| `1` | Forward strand (same orientation as chromosome coordinates — "sense" strand) |
| `-1` | Reverse strand (complementary strand — must be reverse-complemented to read) |

This field is relevant when interpreting splice site sequences or when integrating with
strand-specific data (e.g., RNA-seq). It does not affect the variant coordinates, which are
always reported on the forward strand regardless of gene orientation.

### 2f. Population Allele Frequencies — gnomAD Genomes v4.1

**gnomAD (Genome Aggregation Database)** aggregates sequencing data from large cohorts of
individuals who are not selected for disease (population reference). Version 4.1 includes over
730,000 whole genomes.

The **allele frequency (AF)** for a variant in gnomAD represents how often that alternate allele
appears across all chromosomes in the reference population. A high AF means the variant is
common; a low or absent AF means it is rare or novel.

In rare disease genetics, gnomAD frequencies are the primary tool for **distinguishing
pathogenic variants (rare) from benign polymorphisms (common)**. The underlying assumption is
that variants causing severe Mendelian disease must be rare in the general population because
strong negative selection prevents them from becoming common.

---

#### `gnomADg`
**Type:** string | None

The internal gnomAD variant identifier / key for this position. Used to uniquely identify the
entry in the gnomAD database. `None` if the variant is absent from gnomAD (novel variant).

---

#### `gnomADg_AF`
**Type:** string | None

The **global allele frequency** across all gnomAD Genomes v4.1 samples, pooling all populations.
Stored as a string (e.g., `"0.0032"`) — convert to float for numerical operations.

`None` means the variant was **not observed** in gnomAD — it is absent from >730,000 genomes.
This is a strong signal for rarity and is informative in rare disease analyses.

---

#### Population-specific AF columns

Each column below gives the allele frequency within a specific **population ancestry group**.
Using population-matched frequencies is critical because a variant can be common in one
population and absent from another, and using the global AF would mask this.

| Column | Population | Approx. n genomes (v4.1) | Notes |
|--------|------------|--------------------------|-------|
| `gnomADg_AF_afr` | African / African-American | ~76,000 | Highest genetic diversity; contains many variants rare elsewhere |
| `gnomADg_AF_ami` | Amish | ~500 | Small founder population; high rates of recessive disease |
| `gnomADg_AF_amr` | Latino / Admixed American | ~45,000 | Mixed Indigenous American + European + African ancestry |
| `gnomADg_AF_asj` | Ashkenazi Jewish | ~10,000 | Founder population; elevated carrier frequency for several recessive disorders |
| `gnomADg_AF_eas` | East Asian | ~10,000 | Includes Chinese, Japanese, Korean, Vietnamese individuals |
| `gnomADg_AF_fin` | Finnish | ~24,000 | Founder population with a distinct disease spectrum (Finnish disease heritage) |
| `gnomADg_AF_nfe` | Non-Finnish European | ~400,000 | Largest gnomAD subgroup; dominates the global AF |
| `gnomADg_AF_sas` | South Asian | ~16,000 | Includes Indian, Pakistani, Sri Lankan, and Bangladeshi individuals |

All values are strings in the parsed data; convert with `pd.to_numeric(df["gnomADg_AF_nfe"], errors="coerce")`.

**Why population-specific AF matters:**
If your cohort is predominantly of South Asian ancestry, filter on `gnomADg_AF_sas` rather than
`gnomADg_AF` to avoid retaining variants that are common in that population but rare globally
(which would inflate false positives), or conversely, to avoid filtering out variants that are
rare in the study population but appear common because of the large NFE group in global AF.

## 3. Sample / Genotype Fields (FORMAT)

These fields describe the **genotype call** for the individual patient at each variant site.
They are produced by the variant caller (DeepVariant v1.6.0) based on the raw sequencing reads.
Understanding these fields is essential for assessing call quality and correctly classifying
each variant as heterozygous or homozygous.

---

#### `GT` — Genotype
**Type:** string | None

The called genotype, expressed as two allele indices separated by `/` (unphased) or `|` (phased,
when haplotype information is available). Index `0` always refers to the REF allele; `1` refers
to the first ALT allele; `2` would refer to a second ALT allele at a multi-allelic site.

| GT | Zygosity | Meaning |
|----|----------|---------|
| `0/0` | Homozygous reference | Both copies carry the reference allele — no variant |
| `0/1` | Heterozygous | One copy has REF, the other has ALT |
| `1/1` | Homozygous alternate | Both copies carry the alternate allele |
| `1/2` | Compound heterozygous | Two different alternate alleles at the same site |
| `./.` | No call | The caller could not determine the genotype (e.g., insufficient depth) |
| `./1` | Partial no-call | One allele could not be determined |

In the context of rare Mendelian disease:
- **Dominant disorders**: look for `0/1` (heterozygous) in affected individuals.
- **Recessive disorders**: look for `1/1` (homozygous) or `0/1` in both copies of different
  genes (compound heterozygous), or use burden testing across all rare heterozygous variants in
  the same gene.

---

#### `GQ` — Genotype Quality
**Type:** string | None (integer when converted)

Phred-scaled confidence in the assigned genotype. Specifically, it is –10 × log₁₀ of the
probability that the genotype call is **wrong** (i.e., the true genotype is different from the
one reported in `GT`).

| GQ | Probability GT is wrong |
|----|-------------------------|
| 10 | 10 % |
| 20 | 1 % |
| 30 | 0.1 % |
| 50 | 0.001 % |

Standard thresholds in variant analysis:

| Threshold | Use case |
|-----------|----------|
| `GQ ≥ 20` | Minimum for most analyses |
| `GQ ≥ 30` | Recommended for high-confidence calls (e.g., de novo detection) |

Low GQ variants should be treated as uncertain and typically excluded or flagged.

---

#### `DP` — Read Depth
**Type:** string | None (integer when converted)

The total number of sequencing reads covering this genomic position in the sample. Higher depth
means more statistical power to distinguish real variants from sequencing errors.

Interpretation:

| DP | Interpretation |
|----|----------------|
| < 10 | Too low to call reliably — high risk of allelic dropout |
| 10–20 | Borderline — heterozygous calls may be unreliable |
| 20–100 | Adequate for standard germline variant calling |
| > 100 | High depth — very reliable calls; also common in targeted sequencing |

Minimum DP ≥ 10 is a baseline; DP ≥ 20 is preferred for heterozygous calls because with fewer
reads you may not observe both alleles even if both are truly present.

---

#### `AD` — Allele Depth
**Type:** string | None

Comma-separated read counts for each allele in the order REF, ALT (and further ALTs for
multi-allelic sites). For example, `AD = "60,60"` means 60 reads support the REF and 60 reads
support the ALT.

`AD` is the numerator breakdown of `DP`, though `sum(AD)` may differ slightly from `DP` if
some reads were excluded from allele-specific counting due to low mapping quality.

Use cases:
- Calculate VAF manually: `AD_alt / (AD_ref + AD_alt)`
- Detect allele balance issues: extreme imbalance (e.g., `90,5` for a `GT=0/1` call) may
  indicate a sequencing artefact or loss of heterozygosity.

---

#### `VAF` — Variant Allele Fraction
**Type:** string | None (float when converted)

The fraction of reads supporting the alternate allele: `AD_alt / DP`. Values range from 0.0
to 1.0.

VAF is the most direct indicator of the biological origin of the variant and its zygosity:

| VAF range | Interpretation in germline sequencing |
|-----------|---------------------------------------|
| 0.00–0.10 | Likely sequencing artefact or very low-level somatic mosaic — typically excluded |
| 0.10–0.35 | Low-level mosaic variant (e.g., postzygotic mutation); low-confidence heterozygous |
| 0.35–0.65 | **Expected range for a germline heterozygous variant** |
| 0.65–0.90 | Skewed allele balance — possible loss of heterozygosity, technical bias, or copy number gain |
| 0.90–1.00 | **Expected range for a germline homozygous alternate variant** |

In rare disease research, variants with `VAF < 0.2` are commonly excluded to reduce false
positives, unless the study specifically targets somatic mosaicism.

---

#### `PL` — Phred-scaled Genotype Likelihoods
**Type:** string | None

Comma-separated Phred-scaled likelihoods for each **possible genotype** at this site, in the
order: homozygous reference (RR), heterozygous (RA), homozygous alternate (AA).

The called genotype (`GT`) always has a PL of 0 (most likely). Other genotypes have positive
PL values; the higher the value, the less likely that genotype.

Example: `PL = "0,60,900"`
- P(GT = 0/0) is most likely (PL = 0)
- P(GT = 0/1) is 10⁶ times less likely (PL = 60 → 1-in-1,000,000 probability of being wrong)
- P(GT = 1/1) is extremely unlikely (PL = 900)

In practice, PL values are used by downstream variant genotypers and joint-calling pipelines
(e.g., GATK GenotypeGVCFs) for re-genotyping, and are less often filtered on directly.
A large difference between the lowest and second-lowest PL (high GQ) indicates a confident call.

---

## 4. Filtering Recommendations for Rare Disease GWAS

A standard, stepwise filtering cascade for a rare disease cohort. Apply filters in this order
to maximise sensitivity while controlling for technical and population noise.

### Step 1 — Remove low-quality calls (technical filters)

```python
df = df[df["FILTER"] == "PASS"]
df = df[pd.to_numeric(df["DP"], errors="coerce") >= 10]
df = df[pd.to_numeric(df["GQ"], errors="coerce") >= 20]
df = df[pd.to_numeric(df["VAF"], errors="coerce") >= 0.2]
```

### Step 2 — Focus on protein-coding transcripts

```python
df = df[df["BIOTYPE"] == "protein_coding"]
```

### Step 3 — Retain variants with predicted functional impact

```python
# Keep HIGH and MODERATE impact variants
df = df[df["IMPACT"].isin(["HIGH", "MODERATE"])]

# Optionally add splice-region variants (LOW impact but potentially pathogenic)
# df = df[df["IMPACT"].isin(["HIGH", "MODERATE"]) | df["Consequence"].str.contains("splice", na=False)]
```

### Step 4 — Filter by population allele frequency (rarity filter)

```python
af = pd.to_numeric(df["gnomADg_AF"], errors="coerce")

# Dominant disease model: AF < 0.1 %
df_dominant  = df[af.isna() | (af < 0.001)]

# Recessive disease model: AF < 1 %
df_recessive = df[af.isna() | (af < 0.01)]
```

For population-matched filtering, replace `gnomADg_AF` with the ancestry-specific column
(e.g., `gnomADg_AF_nfe` for a predominantly European cohort).

### Step 5 — Exclude variants in pseudogenes / poor-quality regions (optional)

```python
# Exclude transcripts with annotation flags
df = df[df["FLAGS"].isna()]

# Exclude known pseudogene biotypes
pseudogene_types = ["processed_pseudogene", "unprocessed_pseudogene", "transcribed_pseudogene"]
df = df[~df["BIOTYPE"].isin(pseudogene_types)]
```

### Recommended thresholds summary

| Parameter | Minimum threshold | Stringent threshold |
|-----------|-------------------|---------------------|
| `DP` | ≥ 10 | ≥ 20 |
| `GQ` | ≥ 20 | ≥ 30 |
| `VAF` | ≥ 0.20 | ≥ 0.35 |
| `gnomADg_AF` (dominant) | < 0.001 | < 0.0001 |
| `gnomADg_AF` (recessive) | < 0.01 | < 0.001 |

## 6. Glossary

| Term | Definition |
|------|------------|
| **VCF** | Variant Call Format — the standard plain-text file format for storing DNA sequence variants relative to a reference genome |
| **VEP** | Variant Effect Predictor — Ensembl's tool for annotating the functional consequences of variants |
| **CSQ** | Consequence — the INFO sub-field that VEP populates with pipe-delimited annotations |
| **DeepVariant** | Google's deep-learning-based short-read germline variant caller, used to call variants from aligned reads |
| **Sarek** | nf-core Nextflow bioinformatics pipeline for germline and somatic variant detection and annotation |
| **GRCh38 / hg38** | Genome Reference Consortium Human genome assembly version 38 — the current standard human reference genome |
| **gnomAD** | Genome Aggregation Database — the largest publicly available catalogue of human genetic variation, derived from population sequencing studies |
| **gnomADg** | gnomAD Genomes sub-dataset (whole-genome sequencing), as opposed to gnomADe (whole-exome sequencing) |
| **Allele Frequency (AF)** | The proportion of chromosomes in a population carrying a specific allele at a given position; ranges from 0 to 1 |
| **Minor Allele Frequency (MAF)** | The frequency of the less common allele; for rare disease work, the focus is on variants with very low MAF |
| **Phred score** | A logarithmic quality metric: Q = –10 × log₁₀(P[error]). Q20 = 1 % error rate; Q30 = 0.1 % error rate |
| **HGVS** | Human Genome Variation Society — standards body that defines the nomenclature for describing genetic variants (HGVSc, HGVSp) |
| **SO (Sequence Ontology)** | A controlled, hierarchical vocabulary for describing genomic features and variant effects |
| **HGNC** | HUGO Gene Nomenclature Committee — the authority for assigning standardised human gene symbols |
| **rsID** | Reference SNP cluster ID — a unique identifier from dbSNP for a known variant position |
| **Zygosity** | Whether an individual carries one (heterozygous) or two (homozygous) copies of a variant allele |
| **VAF** | Variant Allele Fraction — fraction of sequencing reads at a position that support the alternate allele |
| **LOH** | Loss of Heterozygosity — loss of one chromosomal copy, converting a heterozygous site to apparent homozygosity |
| **NMD** | Nonsense-Mediated Decay — cellular quality-control mechanism that degrades mRNAs containing premature stop codons |
| **ACMG/AMP** | American College of Medical Genetics / Association for Molecular Pathology — guidelines for classifying variant pathogenicity as Pathogenic, Likely Pathogenic, VUS, Likely Benign, or Benign |
| **Burden test** | A statistical test that aggregates rare variant counts within a gene across a cohort to detect association with disease (e.g., SKAT, CMC, ACAT) |
| **De novo variant** | A variant present in a child but absent from both biological parents — important in dominant developmental disorders |
| **Mendelian disease** | A disease caused by a single gene defect following Mendelian inheritance patterns (dominant, recessive, X-linked) |
| **Compound heterozygous** | Two different pathogenic variants in the two copies of the same gene — each inherited from one parent — together causing a recessive disorder |
